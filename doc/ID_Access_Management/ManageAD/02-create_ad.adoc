= 2-2. Azure AD テナントを作成する

この文章は、必要な**環境準備**と、Azure Active Directory (Azure AD) の**テナント**に関する基本的な説明をしています。

== 📌 要点のまとめ

=== **環境要件:**

* コースの演習には、**Azure AD テナント**と**グローバル管理者権限**が必要。
* 組織のテナントで権限の低いユーザーである場合は、設定変更やユーザー追加などを自由に行える**個人のテスト用テナント**を作成することが推奨される。

=== *テナントの作成:*

* Azure ポータルで「テナントの管理」から簡単に作成できる（費用はかからない）。
* 一意のドメイン名（初期は `onmicrosoft.com` ドメイン）と、データ保存場所に関わる*リージョン*を選択する。

=== *テナントの切り替え:*

* 1つのユーザーアカウントで複数のテナントに所属でき、「テナントの管理」からアクティブなテナントを切り替えることができる。

=== *サブスクリプションと制限:*

* 新しく作成したテナントは**無料プラン**であり、初期状態では Azure リソースを作成するための**サブスクリプション**が付いていない。
* Azure AD の基本的な操作（ユーザー/グループ作成など）はサブスクリプションなしでも可能。
* 条件付きアクセスなどの高度な機能には、上位プランへのアップグレードが必要になる（これについては後述）。

=== *コースのスコープ:*

* このコース（AZ-500、セキュリティ試験）では、ID とアクセス管理に特化した SC-300 試験ほど Azure AD のすべてを深くは扱わず、**セキュリティ**に関連する基本的な要素（サービスプリンシパル、ユーザーIDなど）に焦点を当てて進められる。
